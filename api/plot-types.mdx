## PlotLine

Renders a line chart by connecting data points with straight lines.

### Props

<ParamField path="xc" type="ChartXControl" required>
  X-axis control for horizontal positioning
</ParamField>

<ParamField path="yc" type="ChartYControl" required>
  Y-axis control for vertical positioning
</ParamField>

<ParamField path="tvar" type="TVar<PineData[]>" required>
  Time series variable containing the data
</ParamField>

<ParamField path="name" type="string" required>
  Name of the plot
</ParamField>

<ParamField path="atIndex" type="number" required>
  Index of the data in the PineData array to plot
</ParamField>

<ParamField path="options" type="PlotOptions" required>
  Plot styling options including color, linewidth, fillgaps, etc.
</ParamField>

<ParamField path="depth" type="number">
  Z-index depth for layering (optional)
</ParamField>

### Example

```tsx
import PlotLine from './lib/charting/plot/PlotLine';

<PlotLine
  tvar={indicatorVar}
  name="EMA 20"
  options={{ color: '#2962FF', linewidth: 2, fillgaps: true }}
  atIndex={0}
  xc={xc}
  yc={yc}
  depth={1}
/>
```

### Notes

- Gaps in data are handled based on `options.fillgaps`
- When `fillgaps` is true, connects across missing values
- When `fillgaps` is false, creates separate line segments

## PlotKline

Renders candlestick, bar, or line charts for OHLC (Open-High-Low-Close) data.

### Props

<ParamField path="xc" type="ChartXControl" required>
  X-axis control for horizontal positioning
</ParamField>

<ParamField path="yc" type="ChartYControl" required>
  Y-axis control for vertical positioning
</ParamField>

<ParamField path="kvar" type="TVar<Kline>" required>
  Time series variable containing kline/candlestick data
</ParamField>

<ParamField path="kind" type="KlineKind" required>
  Chart style: `'candle'`, `'bar'`, or `'line'`
</ParamField>

<ParamField path="depth" type="number" required>
  Z-index depth for layering
</ParamField>

### KlineKind Types

<ResponseField name="candle" type="string">
  Traditional candlestick chart with hollow/filled rectangles
</ResponseField>

<ResponseField name="bar" type="string">
  OHLC bar chart with horizontal ticks
</ResponseField>

<ResponseField name="line" type="string">
  Line chart connecting close prices
</ResponseField>

### Example

```tsx
import PlotKline from './lib/charting/plot/PlotKline';

<PlotKline
  kvar={klineVar}
  xc={xc}
  yc={yc}
  kind="candle"
  depth={0}
/>
```

### Rendering Details

**Candle Style**: Renders rectangles for the body (open to close) with wicks extending to high/low. Positive candles (close > open) and negative candles (close < open) use different CSS classes.

**Bar Style**: Renders vertical lines from high to low with horizontal ticks for open (left) and close (right).

**Line Style**: Connects the close prices with a continuous line, showing min/max range when bars are compressed.

## PlotFill

<ParamField path="xc" type="ChartXControl" required>

</ParamField>

<ParamField path="yc" type="ChartYControl" required>

<ParamField path="name" type="string" required>
  Name of the fill
</ParamField>

<ParamField path="options" type="PlotOptions" required>
  Options including plot1, plot2 (indices), color, and fillgaps

<ParamField path="depth" type="number">

<ParamField path="options.plot1" type="number" required>
  Index of the first plot to fill from
</ParamField>

<ParamField path="options.plot2" type="number" required>
  Index of the second plot to fill to
</ParamField>

<ParamField path="options.color" type="string" required>
  Fill color
</ParamField>

### Example

```tsx
import PlotFill from './lib/charting/plot/PlotFill';

<PlotFill
  tvar={indicatorVar}
  xc={xc}
  yc={yc}
  depth={0}
  options={{
    plot1: 0,  // First EMA
    plot2: 1,  // Second EMA
    color: 'rgba(41, 98, 255, 0.1)',
    fillgaps: true
  }}
  name="EMA Fill"
/>
```

## PlotVolume

**Location**: `src/lib/charting/plot/PlotVolume.tsx`

Renders trading volume as a histogram with positive and negative bars based on price movement.

### Props

<ParamField path="xc" type="ChartXControl" required>
  X-axis control for horizontal positioning
</ParamField>

<ParamField path="yc" type="ChartYControl" required>
  Y-axis control for vertical positioning
</ParamField>

<ParamField path="kvar" type="TVar<Kline>" required>
  Time series variable containing Kline (OHLCV) data
</ParamField>

<ParamField path="depth" type="number">

</ParamField>

### Example

```tsx
import PlotVolume from './lib/charting/plot/PlotVolume';

<PlotVolume
  kvar={klineVar}
  xc={xc}
  yc={yc}
  depth={0}
/>
```

### Rendering Details

- **Positive bars** (close >= open): Rendered with `className="positive"`
- **Negative bars** (close < open): Rendered with `className="negative"`
- **Bar width**: Auto-adjusts based on available space
- **Compression**: Displays maximum volume when multiple bars are compressed
- Bars are drawn from zero to the volume value

## PlotDrawingLine

**Location**: `src/lib/charting/plot/PlotDrawingLine.tsx`

Renders dynamic lines from PineScript's line drawing API, supporting lines created at runtime.

### Props

<ParamField path="xc" type="ChartXControl" required>
  X-axis control for horizontal positioning
</ParamField>

<ParamField path="yc" type="ChartYControl" required>
  Y-axis control for vertical positioning
</ParamField>

<ParamField path="tvar" type="TVar<PineData[]>" required>
  Time series variable containing LineObject arrays
</ParamField>

<ParamField path="name" type="string" required>
  Name of the plot
</ParamField>

<ParamField path="atIndex" type="number" required>
  Index of the data in the PineData array
</ParamField>

<ParamField path="options" type="PlotOptions" required>
  Plot options
</ParamField>

<ParamField path="depth" type="number">
  Z-index depth for layering (optional)
</ParamField>

### LineObject Interface

Each line is defined by a LineObject with these properties:

<ResponseField name="id" type="number">
  Unique identifier for the line
</ResponseField>

<ResponseField name="x1" type="number">
  X-coordinate of start point (bar index or time)
</ResponseField>

<ResponseField name="y1" type="number">
  Y-coordinate of start point (price value)
</ResponseField>

<ResponseField name="x2" type="number">
  X-coordinate of end point
</ResponseField>

<ResponseField name="y2" type="number">
  Y-coordinate of end point
</ResponseField>

<ResponseField name="xloc" type="'time' | 'bar_index'">
  Coordinate system for x values
</ResponseField>

<ResponseField name="color" type="string">
  Line color
</ResponseField>

<ResponseField name="style" type="'style_solid' | 'style_dashed' | 'style_dotted'">
  Line style
</ResponseField>

<ResponseField name="width" type="number">
  Line width in pixels
</ResponseField>

### Example

```tsx
// PineScript code that creates dynamic lines
//@version=5
indicator("Trend Lines", overlay=true)

var line trendLine = na
if ta.crossover(close, ta.sma(close, 20))
    trendLine := line.new(bar_index, low, bar_index + 10, high, 
                          color=color.green, width=2)

// Rendered automatically by PlotDrawingLine
```

### Line Styles

- `style_solid`: Continuous line
- `style_dashed`: Dashed line (4px dash, 3px gap)
- `style_dotted`: Dotted line (1px dot, 2px gap)

### Notes

- Supports both time-based and bar-index-based coordinates
- Multiple lines can share the same ID (will be merged)
- Used by PineScript's `line.new()` function
